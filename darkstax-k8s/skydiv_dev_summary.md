
Skydive-UI is developed by the Skydive open-source community, led historically by engineers focused on network observability, cloud infrastructure, and Kubernetes visibility. The project is designed as a lightweight, standalone web application that visualizes graph-based topology data produced by the Skydive analyzer. The developers’ intent is to keep the UI decoupled from data collection and analytics logic, allowing it to function both with a live backend and with static topology snapshots.

From a technology standpoint, Skydive-UI is a client-side JavaScript application built primarily using **AngularJS (1.x)** as the core framework. AngularJS provides the MVC structure, dependency injection, routing, and UI state management. The UI relies heavily on **D3.js** for rendering and interacting with graph-based visualizations, including hierarchical layouts, node expansion, edge rendering, and visual styling based on metadata (status, type, category). SVG is the primary rendering medium, enabling scalable and interactive topology diagrams.

The UI consumes topology data formatted as a **Skydive graph JSON**, which contains nodes (vertices) and edges, each enriched with metadata fields. These nodes represent Kubernetes objects, hosts, containers, networks, and other infrastructure components, while edges represent ownership, containment, and logical or network relationships. The JSON can be fetched dynamically from a running Skydive analyzer over HTTP/WebSocket APIs or loaded statically via a URL parameter, as demonstrated by the Kubernetes demo topology.

Internally, the application is organized into Angular controllers, services, and directives. Services handle data loading, query execution, and interaction with the Skydive API (or static JSON loader). Controllers manage UI state such as selected nodes, expanded elements, filters, and highlights. Custom directives encapsulate reusable UI components like the topology canvas, metadata panels, and control widgets. The graph rendering pipeline transforms raw topology data into a visual graph model that supports zooming, panning, expanding, collapsing, and dynamic highlighting.

Skydive-UI supports **Gremlin-style graph queries** on the client side, allowing users to filter, highlight, or focus on subsets of the topology. These queries are either executed remotely against the Skydive analyzer or applied locally to static graph data. This design allows developers and operators to explore large Kubernetes environments interactively without reloading or rebuilding the entire graph.

Styling and layout are handled using standard web technologies, including HTML, CSS, and Angular templates. Node appearance (icons, colors, shapes) is driven by metadata such as resource type, category, and status. For Kubernetes, this enables immediate visual differentiation between clusters, namespaces, pods, services, policies, and storage objects, as well as quick identification of unhealthy resources.

In its current form, Skydive-UI operates as a **pure visualization and interaction layer**. It does not perform data collection, correlation, or analytics itself. Instead, it depends entirely on the Skydive backend (or a static snapshot) to supply a precomputed graph model. This separation of concerns keeps the UI simple, portable, and easy to extend, while allowing advanced graph construction and enrichment to evolve independently in the backend.

Overall, Skydive-UI reflects a developer philosophy centered on modularity, graph-first visualization, and backend independence. It combines AngularJS for application structure, D3.js for topology rendering, and Skydive’s graph schema to deliver an interactive, extensible Kubernetes and infrastructure topology viewer that works both live and offline.
